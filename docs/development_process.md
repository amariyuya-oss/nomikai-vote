## 3. Phase1: MVP開発

### 開発順序

1. **F01** 飲み会ルーム作成（幹事フロー）
2. **F02** ルーム参加（URL共有・参加者フロー）
3. **F03** 帰宅希望時間投票（匿名・1デバイス1票）
4. **F04** 二次会参加投票（匿名・1デバイス1票）
5. **F05** 投票結果リアルタイム集計・表示
6. **F08** ルーム自動期限（24時間）

### 完了条件

- スマホブラウザ（iOS / Android Chrome）で全フローが通ること
- 投票後3秒以内に結果へ反映されること
- 同一デバイスからの二重投票が防止されること

---

## 4. 開発ルール

- ブランチ: `main`（本番）/ `dev`（開発）/ `feature/xxx`（機能単位）
- コミット粒度: 機能単位で1コミット
- 技術スタック確定後に追記する
