# B07 広告配置設計書

作成日: 2026-06-07

---

## 1. 方針

マネタイズ方針（要件定義書 第8章）に従い、Phase1はGoogle AdSenseを使用する。広告枠は設計段階から確保し、後付けによるUIレイアウト崩壊を防ぐ。

---

## 2. 広告配置画面一覧

| 画面 | 配置箇所 | 広告タイプ | 理由 |
|------|---------|-----------|------|
| P04 投票完了画面 | 完了メッセージの下 | バナー（320×50） | 投票完了後のユーザーの離脱前に表示。滞在時間が発生するタイミング |
| P05 結果表示画面 | 集計結果の下 | バナー（320×50） | 結果確認のために繰り返し訪れる画面。滞在時間が長い |

---

## 3. 広告非配置画面

| 画面 | 理由 |
|------|------|
| P01 ルーム作成画面 | 幹事の操作に集中させる。離脱リスクを排除 |
| P02 ルーム作成完了画面 | URLシェアのCVRを最大化する。広告による視線の分散を避ける |
| P03 参加者投票画面 | 投票完了のCVRを最大化する |
| P06 幹事管理画面 | P05と同一内容。幹事用途のため広告なし |
| P07 期限切れ画面 | ネガティブな体験に広告を重ねない |

---

## 4. 広告枠の実装方針

- Google AdSenseのスクリプトを `index.html` に読み込む
- 各広告配置箇所に固定サイズのdiv要素を事前確保する（コンテンツのレイアウトシフトを防ぐ）
- 広告読み込み失敗時はdiv要素を非表示にし、レイアウトが崩れないようにする

---

| フェーズ | 広告施策 |
|---------|---------|
| Phase2 | タクシーアプリ（GO・Uber等）とのタイアップ広告。終電情報表示画面に配置し、文脈一致を図る |
| Phase3 | 二次会会場送客（ぐるなび・食べログ等）への誘導バナー |
