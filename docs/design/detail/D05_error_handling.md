## 1. エラー分類

| 分類 | 説明 |
|------|------|
| バリデーションエラー | クライアント側の入力値不正 |
| APIエラー | Edge Function や Supabase REST API からのエラーレスポンス |
| DBエラー | RLS 違反・制約違反など |
| ネットワークエラー | 通信タイムアウト・オフライン |
| 状態エラー | 想定外の画面アクセス（sessionStorage欠損など） |

## 3. React Router v7 errorElement の活用

loader 関数内でエラーが発生した場合（ルーム不存在・DB接続失敗など）は、コンポーネント内でのエラー表示ではなく React Router v7 の `errorElement` で捕捉する。

| エラー # | loader で throw する条件 | errorElement での表示 |
|---------|------------------------|----------------------|
| E11 | rooms が 0 件（participantToken 不一致） | 「このURLは無効です。URLをご確認ください」 |
| E19 | rooms が 0 件（participantToken 不一致） | 「このURLは無効です。URLをご確認ください」 |

loader 内の実装イメージ:
```ts
// /room/:participantToken のloader
export async function loader({ params }) {
  const { data } = await supabase
    .from('rooms')
    .select('...')
    .eq('participant_token', params.participantToken)
    .single()
  if (!data) throw new Response('Not Found', { status: 404 })
  return data
}
```

ルーム期限切れ（E12・E20）は loader で throw せず、コンポーネント側で `expires_at < NOW()` を判定して P07 をインライン表示する（期限切れは「無効URL」とは意味が異なるため）。

## 4. 共通エラー処理方針

### バリデーションエラー
- submit 時に全フィールドを一括チェック
- エラーがある場合はフォームの先頭フィールドのエラーにスクロール
- フィールド個別には onBlur 後にリアルタイム検証

### APIエラー・ネットワークエラー
- 画面上部に固定バナーで表示
- 操作中のボタンは無効化を解除し、再試行できるようにする
- 自動リトライは行わない（ユーザーによる再試行を基本とする）

### エラー表示のクリア
- バナー系エラーは次の操作（ボタン押下等）時にクリアする
- フィールドエラーは入力変更時にクリアする

## 4. ログ方針

MVP フェーズでは詳細なエラーログ収集は行わない。Edge Function の実行ログは Supabase Dashboard で確認可能。クライアントサイドのエラーは `console.error` のみ（外部モニタリングは Phase2 以降に検討）。
